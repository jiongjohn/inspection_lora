# Inspection LoRA — 训练总结报告

> 项目: Prometheus 指标数据 AI 巡检 LoRA 微调
> 模型: Qwen2.5-1.5B-Instruct + MLX LoRA
> 日期: 2026-02-26

---

## 1. 项目概述

基于 Qwen2.5-1.5B-Instruct 模型，使用 MLX LoRA 在 Apple Silicon (macOS) 上进行微调，使模型能够分析 Prometheus 监控指标数据并生成结构化的中文巡检报告。

## 3. 超参调优实验

三组 LoRA rank 实验，其余超参保持一致：

| 超参 | 值 |
|------|------|
| batch_size | 2 |
| grad_accumulation | 8 (有效 batch=16) |
| learning_rate | 2e-4 |
| lr_schedule | cosine decay (warmup=50) |
| iters | 1,000 |
| max_seq_length | 2,048 |
| dropout | 0.05 |
| target_modules | q/k/v/o_proj + gate/up/down_proj |
| mask_prompt | true |

### 实验结果

| 指标 | rank=8 | rank=16 | rank=32 |
|------|--------|---------|---------|
| Train Loss | 0.097 | 0.099 | 0.105 |
| Val Loss | 0.099 | 0.098 | 0.102 |
| Peak Memory | 5.40 GB | 5.62 GB | 5.97 GB |
| 训练速度 (It/sec) | 0.222 | 0.228 | 0.215 |
| Tokens/sec | 86.3 | 88.3 | 83.4 |

### 分析

- **rank=8**: 最低 train loss (0.097)，内存最省 (5.40 GB)，val loss 0.099
- **rank=16**: 最低 val loss (0.098)，训练速度最快 (0.228 it/s)，泛化最优
- **rank=32**: loss 最高 (0.105/0.102)，内存最大 (5.97 GB)，速度最慢，出现轻微过拟合迹象

**结论: rank=16 为最优配置**，在泛化能力 (最低 val loss)、训练效率和内存占用之间取得最佳平衡。rank=8 接近但略逊，rank=32 参数过多反而导致性能下降。

## 4. 模型评估 (rank=16)

### 4.1 微调模型 vs 基座模型

| 指标 | 基座模型 | 微调模型 | 变化 | MVP 标准 | 状态 |
|------|---------|---------|------|---------|------|
| 异常检测 F1 | 50.00% | 95.44% | +45.44% | > 75% | ✅ |
| 数值准确率 | 29.84% | 83.72% | +53.88% | > 90% | ⚠️ |
| 假阳性率 | 35.21% | 0.00% | -35.21% | < 15% | ✅ |
| 结构完整性 | 56.18% | 97.27% | +41.09% | — | ✅ |

### 4.2 关键提升

- **F1 提升 +45.44%**: 从随机水平 (50%) 提升至近乎完美 (95.44%)
- **FPR 降至 0%**: 基座模型 35.21% 误报率完全消除
- **结构完整性 +41.09%**: 输出格式从半结构化提升至高度规范
- **数值准确率 +53.88%**: 大幅提升但未达 90% 门槛 (83.72%)

### 4.3 数值准确率分析

数值准确率 83.72% 未达 90% MVP 标准，原因分析：
- 模型生成的百分比数值与 ground truth 存在微小偏差（容差 ±5%）
- 895/1069 个数值匹配，174 个超出容差
- 这是生成式模型的固有特性——精确数值复现比语义理解更难
- 实际应用中，趋势判断和异常识别比精确数值更重要

## 8. 结论

1. **微调效果显著**: 所有核心指标大幅超越基座模型，F1 从 50% → 95.44%
2. **rank=16 最优**: 三组超参实验中泛化最好、效率最高
3. **端到端验证通过**: 10/10 真实场景全部通过，平均 4.2/5.0
4. **资源友好**: 峰值仅 5.62 GB 内存，适合 Apple Silicon 本地训练
5. **改进方向**: 数值准确率可通过增加训练数据中的数值多样性、或后处理校正来提升
